# Quickstart: Scales Generation

**Feature**: 001-scales-generation | **Date**: 2026-03-14

---

## Generating Scale Files

Scale files are generated by a Python script committed to `scripts/generate_scales.py`. Run this once to produce all 48 `.mxl` files in `scores/scales/`. The generated files are committed to the repository as static assets.

### Prerequisites

- Python 3.x (no third-party packages required; uses `zipfile` and `xml.etree.ElementTree` from stdlib)

### Run the generator

```bash
# From the repo root
python3 scripts/generate_scales.py

# Output: scores/scales/{48 .mxl files}
```

### Verify the output

```bash
ls scores/scales/ | wc -l
# Expected: 48

# Run generator tests
python3 -m pytest tests/scripts/test_generate_scales.py -v
# or, without pytest:
python3 tests/scripts/test_generate_scales.py
```

## How the Dialog Picks Up Scale Scores

1. `scripts/generate_scales.py` writes `.mxl` files to `scores/scales/`
2. `frontend/public/scores` is a symlink to `../../scores`, so the files are served at `{BASE_URL}scores/scales/{filename}` automatically — no Vite config change required
3. `frontend/src/data/preloadedScores.ts` exports `SCALE_SCORE_GROUP` (hardcoded manifest of all 48 files) and `PRELOADED_CATALOG` (combines ungrouped + scale group)
4. `LoadScoreDialog` renders ungrouped scores via `PreloadedScoreList`, then each group via `ScoreGroupList`
5. `ScoreGroupList` uses a native `<details>/<summary>` element — collapsed by default

---

## Adding a New Score Group in the Future

To add a new subfolder group (e.g., `scores/arpeggios/`):

1. Add files to the new subfolder (manually or via a generator script)
2. In `preloadedScores.ts`, define a new `ScoreGroup` constant with the score manifest
3. Append the new group to `PRELOADED_CATALOG.groups`
4. Write a `ScoreGroupList` test for the new group
5. No component changes needed — `LoadScoreDialog` iterates `PRELOADED_CATALOG.groups` dynamically

## Scale File Naming Convention

```
{Root}_{type}_oct{N}.mxl
```

- `Root`: note name using `b` for flat (e.g., `Bb`, `Db`) and `s` for sharp (e.g., `Fs`, `Cs`)
- `type`: `major` or `minor`
- `N`: octave number (`4` or `5`)

**Examples**: `C_major_oct4.mxl`, `Bb_minor_oct5.mxl`, `Fs_major_oct4.mxl`

---

## Display Name Convention

```
{Root} {Type} — Octave {N}
```

- Flat symbol: `♭` (Unicode U+266D)
- Sharp symbol: `#`

**Examples**: `"C Major — Octave 4"`, `"B♭ Minor — Octave 5"`, `"F# Major — Octave 4"`
