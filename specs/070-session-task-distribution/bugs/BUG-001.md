# Bug Report: Tasks Created for Hands with Only Silent Measures

**Type**: Spec gap  
**Severity**: Medium  
**Feature**: `070-session-task-distribution`  
**Reported**: 2026-06-07  
**Status**: Patched  
**Patched**: 2026-06-07

## Description

When creating tasks to reach a goal to play a score in the Sessions plugin, if the phrase range for a given hand (staffIndex) contains only **silent measures** — measures with no notes to play — the task for that hand is still created. This results in unnecessary practice tasks that have no playable content.

**Steps to reproduce**:
1. Load a multi-staff (piano) score where one staff (e.g., Left Hand) has one or more phrases that contain only rests/no notes (e.g., a score where the LH is silent for an entire phrase region).
2. Open the Sessions plugin → Goals tab.
3. Create a "Learn Score" goal for that score.
4. Observe: RH, LH, and BH tasks are created for every phrase, including the phrase(s) where the LH is entirely silent.

**Expected**: Tasks for a given hand (staffIndex) MUST NOT be created when all measures within the phrase range are silent for that staff — i.e., when `getRegionDifficulty()` returns `null` for that (startMeasure, endMeasure, staffIndex) combination.

**Actual**: The `createGoal()` function in `plugins-external/sessions-plugin/goalEngine.ts` generates tasks for all `taskDefs` (RH, LH, BH) unconditionally. When `getRegionDifficulty()` returns `null` (which the Rust backend emits when `bar_densities.is_empty() || !has_any_note`), the code falls back to medium difficulty (`rawDifficulty?.level ?? 2`) and still creates the task.

**Relevant code** (`plugins-external/sessions-plugin/goalEngine.ts`, `createGoal()`, lines ~110–140):

```typescript
const phraseTasks: SessionTask[] = taskDefs.map(({ staffIndex }) => {
  const rawDifficulty = input.getRegionDifficulty?.(startMeasure0, endMeasure0, staffIndex);
  const difficulty = rawDifficulty?.level ?? 2;  // ← falls back to medium; does NOT skip the task
  const estimatedDurationSecs = estimateTaskDuration(numMeasures, loopCount, difficulty, minResult);

  const task: SessionTask = {
    id: crypto.randomUUID(),
    // ... fields ...
    staffIndex,
    difficulty,
    estimatedDurationSecs,
  };

  return task;  // ← task always returned, even when rawDifficulty is null (silent region)
});
```

The Rust function `compute_region_difficulty` in `backend/src/domain/difficulty/density.rs` explicitly returns `None` (→ TypeScript `null`) when a region has no notes (`!has_any_note`). This `null` signal is the correct mechanism to detect silent regions, but `goalEngine.ts` does not use it as a skip condition.

## Root Cause Analysis

The bug is a **spec gap**: FR-001 was written with the implicit assumption that every hand always has notes in every detected phrase of a piano score. This is true for most well-structured piano scores but fails for scores where one staff is tacet (silent) over extended phrase regions — e.g., a score opening with a solo right-hand melodic line before the left hand enters, or a score where the left hand rests for an entire phrase.

The implementation faithfully followed the spec but did not anticipate this edge case. The Rust backend already provides the correct signal (`compute_region_difficulty` returns `None`/`null` for silent regions via `!has_any_note`), but `goalEngine.ts` treats `null` purely as a "difficulty unknown" fallback (defaulting to medium) rather than as "no notes — skip this task."

No earlier feature spec (061, 067, 068) addressed this either, as they all operated with single-phrase, single-session goals.
