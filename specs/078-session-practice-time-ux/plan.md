# Implementation Plan: Session & Practice Goal Execution UX Improvements

**Branch**: `078-session-practice-time-ux` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)
**Worktree**: `../worktrees/078-session-practice-time-ux`
**Input**: Feature specification from `/specs/078-session-practice-time-ux/spec.md`

## Summary

Three targeted UX improvements to the Sessions and Practice plugins:
1. **Task invested/estimated time display** — each task row shows accumulated practice time alongside the estimate (pure UI, reads existing `practiceTimeMs` + `estimatedDurationSecs`).
2. **Session completion time diff** — closed sessions display a real-vs-estimated summary with a signed delta, styled by overrun/saving; stored as `totalRealTimeSecs` at close time.
3. **Loop count lock in Results overlay** — when practice is launched from a session task, the loop count slider in `ResultsOverlay` is disabled; a tooltip indicates the task controls the count.

No new data capture or schema migration. All values are derived from fields introduced in features 060–071.

## Technical Context

**Language/Version**: TypeScript 5+ / React 18+  
**Primary Dependencies**: React (sessions-plugin external, practice-view-plugin internal), Vitest + React Testing Library (tests), i18next-style locale JSON (i18n)  
**Storage**: IndexedDB — `SessionIndexEntry` gains one optional field (`totalRealTimeSecs`); no schema migration required  
**Testing**: Vitest + RTL for components; Vitest unit tests for `sessionStorage.ts` / `useSessionManager.ts`  
**Target Platform**: Tablet PWA (Chrome/Safari), offline-first  
**Project Type**: web — external plugin (`plugins-external/sessions-plugin/`) + frontend internal plugin (`frontend/plugins/practice-view-plugin/`)  
**Performance Goals**: All derived computations are O(n) over task count (≤50 tasks/session); no async work on the render path  
**Constraints**: Must not regress standalone (non-task) practice flows; `loopCountLocked` has no effect when `undefined` or `false`  
**Scale/Scope**: Per-session: ≤50 tasks; per-task: ≤10 linked practices

## Phase 1 — Design & Contracts

See [data-model.md](data-model.md) and [contracts/component-contracts.ts](contracts/component-contracts.ts).

### Key design decisions

**Story 3 — Loop count lock:**
- Add `loopCountLocked?: boolean` to `ResultsOverlayProps`.
- In `ResultsOverlay.tsx`, render the loop slider with `disabled={loopCountLocked}` and add a `title` attribute (tooltip) equal to the `practice.results.loop_locked_hint` i18n key.
- `PracticeViewPlugin.tsx` passes it as `loopCountLocked={!!taskIdRef.current}`.
- No change to `handleRepractice` — state is already preserved correctly once the slider is locked.

**Story 1 — Invested/estimated in TaskRow:**
- `investedTimeMs` computed inline: `task.linkedPractices.reduce((s, lp) => s + lp.practiceTimeMs, 0)`.
- Display logic:
  - `estimated > 0 && invested >= 0` → `"⏱ Xm / Ym"` (invested / estimated)
  - `estimated = 0 && invested > 0` → `"⏱ Xm invested"`
  - `estimated = 0 && invested = 0` → no time row
- Formatted in minutes via the existing `formatEstimatedDuration` helper (shared).

**Story 2 — Session completion summary:**
- `SessionIndexEntry` gains `totalRealTimeSecs?: number` — optional, backward-compatible.
- `useSessionManager.closeSession()` computes the field from `session.tasks` before saving.
- New `SessionTimeSummary.tsx` pure component — receives `realTimeSecs`, `estimatedTimeSecs?`, `estimatedTaskCount?`, `totalTaskCount?`.
- Rendered inside the collapsed/expanded session detail header for closed sessions in `SessionsPlugin.tsx`.
- Delta styling via CSS modifier classes: `sessions-plugin__time-summary--overrun` and `sessions-plugin__time-summary--saving`.

### Post-design Constitution re-check

All three changes remain UI-layer only with no new async patterns, no layout logic, and tests planned for all acceptance scenarios. No constitution violations introduced by design.

## Implementation Notes

- **No migration needed**: `totalRealTimeSecs` is optional; old closed-session entries simply show the "real only" path or nothing.
- **`SessionTimeSummary` is stateless**: no hooks, no context. Pure `(props) => JSX`.
- **`loopCountLocked` defaults to `undefined`**: falsy, so existing callers of `ResultsOverlay` that don't pass the prop behave identically to today.
- **i18n keys**: All new strings added to both `en.json` and `es.json`. Spanish values may be English text marked with a TODO comment for the translator on first pass.
