# Internal API Contract: WASM Layout Output — Barline Types Extension

**Feature**: 041-repeat-barlines
**Scope**: Rust layout engine output → WASM → TypeScript consumer
**Date**: 2026-06-25

## Context

The WASM layout engine (`compute_layout`) serialises its output as JSON. This contract defines the extended `BarLineType` enum and the new `RepeatDotPosition` type added to `BarLine`. The TypeScript `layout.ts` mirror must match exactly.

---

## Contract 1: Extended BarLineType Enum

**Rust** (`backend/src/layout/types.rs`):
```rust
#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub enum BarLineType {
    Single,
    Double,
    Final,
    RepeatStart,   // NEW
    RepeatEnd,     // NEW
    RepeatBoth,    // NEW
}
```

**TypeScript mirror** (`frontend/src/wasm/layout.ts`):
```typescript
export type BarLineType = 'Single' | 'Double' | 'Final' | 'RepeatStart' | 'RepeatEnd' | 'RepeatBoth';
```

**Constraint**: Serialisation uses `serde` default (PascalCase enum variant names → JSON string). TypeScript literal union mirrors these exact strings.

## Contract 2: RepeatDotPosition Type

**Rust** (`backend/src/layout/types.rs`):
```rust
#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct RepeatDotPosition {
    pub x: f32,       // horizontal centre of the dot, in staff-space units
    pub y: f32,       // vertical centre of the dot, in staff-space units
    pub radius: f32,  // dot radius, in staff-space units (= 0.25 * staff_space)
}
```

**TypeScript mirror** (`frontend/src/wasm/layout.ts`):
```typescript
export interface RepeatDot {
  x: number;
  y: number;
  radius: number;
}
```

## Contract 3: Extended BarLine Type

**Rust** (`backend/src/layout/types.rs`):
```rust
pub struct BarLine {
    pub segments: Vec<BarLineSegment>,
    pub bar_type: BarLineType,
    pub dots: Vec<RepeatDotPosition>,   // NEW — empty for non-repeat types
}
```

**TypeScript mirror** (`frontend/src/wasm/layout.ts`):
```typescript
export interface BarLine {
  segments: BarLineSegment[];
  bar_type: BarLineType;
  dots?: RepeatDot[];  // present and non-empty only for repeat types
}
```

**Constraint**: `dots` is always serialised (never omitted from Rust output), but `LayoutRenderer.tsx` treats an absent or empty `dots` array identically — no dots are rendered for `Single`, `Double`, and `Final` types.

## Contract 4: Dot Geometry Generated by Rust Layout Engine

**File**: `backend/src/layout/mod.rs` — `create_bar_lines()` function.

**Authority**: ALL dot coordinate values are computed in Rust (Constitution Principle VI). TypeScript only reads and renders them.

**Geometry rules**:
```
staff_space     = config.units_per_space  (layout config value, e.g. 10.0 units)
dot_radius      = 0.25 * staff_space
x_offset        = 0.6 * staff_space        // horizontal distance from thick bar centre to dot centre

For RepeatEnd (dots to LEFT of thick bar at x_bar):
  dot_x = x_bar - x_offset
  dot_y[0] = staff_top + 1.0 * staff_space  // 2nd line space (from bottom)
  dot_y[1] = staff_top + 3.0 * staff_space  // 4th line space (from bottom)
  → 2 RepeatDotPosition entries per staff

For RepeatStart (dots to RIGHT of thick bar at x_bar):
  dot_x = x_bar + x_offset
  dot_y[0] = staff_top + 1.0 * staff_space
  dot_y[1] = staff_top + 3.0 * staff_space
  → 2 RepeatDotPosition entries per staff

For RepeatBoth:
  Left dots:  dot_x = x_bar - x_offset, y[0], y[1]
  Right dots: dot_x = x_bar + x_offset, y[0], y[1]
  → 4 RepeatDotPosition entries per staff
```

**Multi-staff constraint**: When a score has multiple staves, dots are generated independently for EACH staff (each staff has its own `staff_top`). The `dots` array on a `BarLine` spans all staves.

## Contract 5: LayoutRenderer.tsx Rendering Rules

**File**: `frontend/src/components/LayoutRenderer.tsx`

**Trigger**: `barLine.dots` is non-empty.

**Action**: For each `dot` in `barLine.dots`, render an SVG `<circle>` at `(dot.x, dot.y)` with `r={dot.radius}`. Fill colour matches the barline stroke colour.

**Constraint**: TypeScript MUST NOT recalculate dot positions. The values in `barLine.dots` are used directly as SVG coordinates. No arithmetic is performed on `dot.x`, `dot.y`, or `dot.radius` in TypeScript (Principle VI).

## Summary: New Output Fields

| Field | Rust type | JSON shape | TS type |
|---|---|---|---|
| `BarLine.bar_type` (extended) | `BarLineType` | `"RepeatStart"` etc. | `BarLineType` string union |
| `BarLine.dots` | `Vec<RepeatDotPosition>` | `[{x, y, radius}, ...]` | `RepeatDot[]` |
| `RepeatDotPosition` | struct | `{x: f32, y: f32, radius: f32}` | `RepeatDot` interface |
