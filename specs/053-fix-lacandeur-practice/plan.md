# Implementation Plan: Fix Practice Issues in La Candeur

**Branch**: `053-fix-lacandeur-practice` | **Date**: 2026-03-23 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/053-fix-lacandeur-practice/spec.md`

## Summary

Seven practice engine bugs in La Candeur (and all scores) are fixed by targeting four layers of the frontend practice plugin:

1. **Duration double-truncation** (`mergePracticeNotesByTick.ts`): Remove the second cross-staff gap clamp that shortens BH-mode note durations. Fixes Bugs 1 (LH chord not green), 2 (M3–M4 truncation), and likely 4 (M15 false requirement) as a cascade.
2. **Highlight persistence after system change** (`LayoutRenderer.tsx`): Add a deferred `reapplyHighlights()` after each SVG rebuild. Fixes Bug 3 (green dot disappears).
3. **Position lockout during practice** (`PracticeViewPlugin.tsx`, `ScoreViewer.tsx`): Gate all navigation handlers behind `isPracticeRunning`. Fixes Bug 6.
4. **Partial results on Stop** (`PracticeViewPlugin.tsx`): Snapshot `noteResults` + `currentIndex` before dispatching `STOP`; render results overlay with partial data. Fixes Bug 7.

Rest enforcement (Bug 5) is addressed through consistent `WRONG_MIDI` handling for inter-onset gap key presses.

No backend or WASM changes. All fixes are frontend-only, confined to the practice plugin and score viewer.
